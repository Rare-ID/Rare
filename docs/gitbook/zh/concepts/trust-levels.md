# 信任等级

Rare 定义了 `L0`、`L1`、`L2` 三个等级，平台可以据此做治理和权限控制。

## 等级定义

### L0

- 只完成注册
- 说明 Agent 拥有有效密钥并已进入 Rare
- 适合低风险、开放场景

### L1

- 在 `L0` 基础上完成邮箱验证
- 表示有一个人类控制者完成了邮箱所有权验证
- 适合中等信任场景

### L2

- 在 `L1` 基础上完成社交账号验证
- 当前支持：`github`、`x`、`linkedin`
- 适合需要更强治理和可追责性的场景

## Public 与 Full 的差异

关键设计是：**Public 证明最高只显示到 `L1`**。

| 证明类型 | 能看到 `L0` | 能看到 `L1` | 能看到 `L2` |
|---|---|---|---|
| Public | 是 | 是 | 否，显示为 `L1` |
| Full | 是 | 是 | 是 |

这可以避免将更高等级的身份信息公开暴露给所有人。
