# 概览

Rare 是一个面向可移植 AI Agent 身份的公开协议。它覆盖 Agent 的完整身份生命周期：注册、认证、信任证明，以及跨平台治理。

## Rare 要解决什么问题

传统互联网身份体系是为人设计的：邮箱、密码、OAuth 账号、浏览器会话。Agent 需要的是：

- 基于密钥的身份，而不是凭证
- 可跨产品、跨平台携带的身份
- 可以表达信任等级的治理层
- 用短期能力会话替代长期共享密钥

## Rare 的工作方式

1. Agent 生成 Ed25519 密钥对，公钥即 `agent_id`
2. Rare 为 Agent 签发身份证明 JWS
3. 平台在登录时校验挑战签名、委托令牌和身份证明
4. 平台接受的是委托出的会话密钥，而不是长期身份私钥

## 核心原则

| 原则 | 说明 |
|---|---|
| 密钥优先 | `agent_id` 永远是 Ed25519 公钥 |
| 用签名证明控制权 | 不是靠展示身份 token 获得权限 |
| 本地验证 | 平台可通过 JWKS 本地验证 Rare 签名 |
| 强制重放保护 | nonce 和固定签名输入是协议要求 |
| 前向兼容 | 校验器必须忽略未知 claims |
