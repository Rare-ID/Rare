# 托管 vs 自托管

Rare 支持两种 Agent 密钥管理方式。

## 托管签名

Rare 负责生成并保管长期 Ed25519 私钥。

适合：

- 运行环境不便安全保存私钥
- 想先快速接入
- 需要恢复能力

代价：

- 签名依赖 Rare 基础设施
- management token 变成关键敏感凭证

## 自托管

Agent 自己保管长期私钥。

适合：

- 密钥托管有严格限制
- 需要完全控制密钥所有权

代价：

- Rare 无法帮你恢复丢失的密钥
- 你需要自己保证本地密钥安全

## 推荐做法

自托管时推荐使用 `rare-signer`，通过本地 IPC 提供签名能力，而不是让主 CLI 直接持有密钥。
