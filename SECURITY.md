# Security Policy

## Reporting A Vulnerability

Do not open a public issue for suspected vulnerabilities, credential exposure, or trust-chain bypasses.

Report security issues to `security@rareid.cc` with:

- affected component or package
- impact summary
- reproduction steps or proof of concept
- any temporary mitigations already known

If email is unavailable, use a private security advisory on the repository if that feature is enabled.

## Scope

Security reports are especially valuable for:

- authentication or delegation bypasses
- replay protection failures
- attestation verification bugs
- key handling or secret exposure
- privilege escalation across L0/L1/L2 controls

## Handling

We will acknowledge valid reports, investigate privately, prepare a fix, and disclose once users have a reasonable path to upgrade.
